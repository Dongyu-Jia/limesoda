# Process: P5 - Implementation Sprint
- **Unique ID**: `P5_IMPLEMENTATION`
- **Goal**: Write code that fulfills the API contracts and Acceptance Criteria.
- **Input**: `P4_ARCHITECTURE` specs + `P2_PRODUCT_REQUIREMENTS` user stories.
- **Output**: Feature branches with committed code and Unit Tests.
- **Criteria**: Code successfully compiles via `tsc` or equivalent tools.
## Small Processes:
1. `P5.1_WRITE_TESTS`: Business Logic Agent writes failing TDD tests.
2. `P5.2_WRITE_CODE`: Business Logic Agent writes implementation to pass tests.
3. `P5.3_COMPILE_CHECK`: Execute deterministic syntax/compile check.
4. `P5.4_OPEN_PR`: Push branch and open GitHub PR.
