# Process: P4 - Architecture Design
- **Unique ID**: `P4_ARCHITECTURE`
- **Goal**: Define math/API contracts to prevent hallucinations.
- **Input**: Approved `P2_PRODUCT_REQUIREMENTS` PRD and `P3_UX_PROTOTYPING` wireframes (if applicable).
- **Output**: Database Schema (e.g., Prisma), OpenAPI specs, and system architecture RFC.
- **Criteria**: Human Tech Lead explicit sign-off on the RFC (Gate 4).
## Small Processes:
1. `P4.1_DRAFT_SCHEMA`: Architect Agent defines DB tables.
2. `P4.2_DRAFT_API`: Architect Agent types out API constraints.
3. `P4.3_FEASIBILITY_CHECK`: EM Agent verifies if the stack can support the design.
4. `P4.4_HUMAN_APPROVAL`: Wait for Tech Lead approval.
